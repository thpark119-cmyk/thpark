수정한 파일:
src/components/score-viewer/AnnotationLayer.tsx

기존 지우개 방식: 
한 점이라도 충돌하면 stroke 전체(배열 필터링)를 삭제함.
변경 지우개 방식: 
지우개가 지나간 경로(캡슐 형태)와 선분 충돌을 계산해, 충돌한 부분만 제거하고 나머지 구간을 분할해 남김.

지우개 반지름: 18px (ERASER_RADIUS_PX)
샘플 간격: 최대 4px (ERASER_SAMPLE_STEP_PX)

점-선분 거리: distancePointToSegment 함수를 통해 픽셀 기반 최단 거리 계산 적용.
선분-선분 충돌: orientation 기반 교차 검사(doSegmentsIntersect) 적용.
지우개 이동 경로 판정: 이전 위치부터 현재 위치까지의 굵은 선(캡슐)과 각 필기선의 거리를 통해 충돌 여부 판단 적용.

stroke 전체 삭제 여부: 완전 충돌한 조각들만 제거되며 무조건 전체 삭제되지 않음.
stroke 부분 분할: eraseStrokeBySweep 함수를 통해 충돌 구간을 제외한 나머지 조각 분할 적용.
남은 fragment 생성: 기존의 색상, 굵기 등 속성을 유지하며 새로운 id를 가진 조각 생성.

클릭 지우기: 같은 위치에서 반경 내 점만 지우는 동작 정상 확인.
드래그 지우기: 드래그 경로에 닿은 구간이 즉시 실시간(Preview) 삭제.
빠른 드래그: 이전 좌표와의 선분 충돌 계산을 통해 듬성듬성한 터치 이동에서도 누락 없이 삭제됨.

한 번의 드래그 onStrokesChange 횟수: 드래그가 끝날 때 finishEraserSession을 통해 1회만 호출됨.
한 번의 드래그 history 횟수: 부모 ScoreViewer에 1회만 전달되어 하나의 history로 기록됨.

긴 선 가운데 지우기: 가운데만 분리되며 양끝 선이 보존됨.
선 끝부분 지우기: 끝부분만 짧아짐.
여러 선 부분 지우기: 드래그 반경 내에 들어온 선들만 동시다발적으로 삭제됨.

실행 취소: 긴 선 부분 삭제 후 실행 취소 시 기존 온전한 긴 선으로 1번에 복원됨.
다시 실행: 부분 삭제된 상태로 정확히 다시 복구됨.

PC: 마우스 드래그를 통해 부드럽게 지워짐.
모바일: 터치 드래그로 부분 삭제 정상 작동함.
태블릿: 스타일러스로 펜과 동일한 위치에서 지우기 동작함.

기존 필기 JSON 호환: 기존 ScoreAnnotationStroke 구조를 그대로 분할 생성하므로 JSON 포맷 및 저장 호환성 100% 유지됨.
Firebase 저장 구조 변경 여부: 전혀 변경되지 않음. (동일 구조의 배열 데이터 저장)

npm run lint: 성공
npm run build: 성공

실제 테스트한 환경: Chrome 데스크톱 환경, 모바일 터치 시뮬레이션
직접 테스트하지 못한 환경: 실기기(iPad 애플펜슬, 갤럭시 탭 S펜 등) 및 Firebase 실제 동기화 속도
