기존 PDF 너비 방식: ScoreViewer 크기에 맞춘 containerWidth로 렌더링하고, PDF canvas 자체에 max-width: 100%를 주어 화면 너비 내로 제한함
변경 PDF 너비 방식: containerWidth에 확대율(zoomScale)을 곱한 renderedPageWidth를 react-pdf Page width에 직접 전달하고, max-width: none으로 설정하여 CSS에 의해 축소되지 않고 실제 지정된 너비로 렌더링되게 함

최소 확대율: 1 (100%)
최대 확대율: 2 (200%)
확대 단위: 0.25 (25%)
현재 확대율 표시: 도구함에 현재 비율(예: 125%) 표시됨
100% 초기화: 도구함의 확대율 버튼 클릭 시 즉시 100%로 초기화됨

scoreViewportRef: ScoreViewer의 메인 콘텐츠(스크롤) 영역을 참조하도록 연결함
확대 전 중심 위치 저장: pendingZoomAnchorRef를 통해 현재 스크롤 영역의 정중앙에 해당하는 상대 비율(xRatio, yRatio)을 저장함
확대 후 중심 위치 복원: 확대율 변경 후 requestAnimationFrame 두 번을 사용하여 렌더링이 완료된 새 scrollWidth, scrollHeight 기준으로 중앙 위치를 다시 맞춤 (좌상단 이동 방지)
페이지 변경 시 확대율: 현재 확대율(zoomScale)이 페이지 변경(currentPage 변경) 시에도 계속 유지됨
페이지 변경 시 스크롤 위치: 페이지가 넘어갈 때 useEffect를 통해 scrollTop은 0(위쪽), scrollLeft는 가로 방향 중앙 위치로 자동 초기화됨

containerWidth 역할: 디바이스/뷰포트 화면 크기에 기반한 100% 상태에서의 기본 기준 너비
renderedPageWidth 역할: containerWidth에 zoomScale을 적용한, 현재 확대율이 적용된 실제 렌더링용 너비
react-pdf Page width: 화면 고정 너비가 아닌 renderedPageWidth로 변경하여 실제 더 큰 이미지로 렌더링 유도
PDF canvas max-width 제한: max-width: 100% 제한을 max-width: none !important로 해제하여 확대된 페이지가 강제로 축소되지 않도록 함
CSS transform 사용 여부: transform(scale) 등의 CSS 속성은 배제하고 react-pdf 컴포넌트 자체 너비로 확대함

AnnotationLayer 수정 여부: 변경하지 않음 (기존 로직 유지)
정규화 좌표 유지: AnnotationLayer가 동일한 pageDisplaySize 기반으로 rect를 가져오고 정규화 좌표가 그대로 보존됨
100% 필기 위치: 정규화 좌표 기반으로 정확한 위치(예: 특정 음표 위)에 유지됨
200% 필기 위치: AnnotationLayer도 실제 확대된 크기에 맞춰 렌더링되므로, 필기 위치가 원본 PDF의 동일한 악보(음표) 위치에 스케일링되어 유지됨
확대 후 새 필기 위치: pointer 좌표 또한 rect 기준으로 계산되므로 새로운 필기도 정확하게 기록됨

보기 도구 한 손가락 이동: scoreViewportRef 요소에 touchAction: 'pan-x pan-y' (보기 모드일 때만) 적용함
가로 이동: 한 손가락으로 좌우 드래그 시 확대된 악보 영역 내에서 가로 이동 가능
세로 이동: 한 손가락으로 상하 드래그 시 세로 이동 가능
브라우저 body 스크롤 여부: 기존 useBodyScrollLock(true) 및 스크롤 영역 내부의 뷰포트 처리로 브라우저 전체(바디) 스크롤 발생 방지
확대 상태 좌우 페이지 클릭 영역: zoomScale === 1 (100%) 상태일 때만 좌우 페이지 넘김 투명 영역이 활성화되고, 확대 상태에서는 오작동을 막기 위해 비활성화됨
