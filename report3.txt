src/utils/scoreAnnotationStorage.ts

기존 필기 삭제 오류 처리: deleteScoreAnnotations 내부에서 try/catch로 오류를 삼키고 console.error만 출력함
변경 필기 삭제 오류 처리: try/catch를 제거하여 실패 시 호출자에게 오류를 전파하도록 수정함

공통 파일 삭제 helper: Promise.allSettled를 사용하는 deleteScoreFileAssets 함수를 Repertoire.tsx에 추가함
원본 파일 삭제: deleteFileFromStorage 호출 (storage/object-not-found는 성공 처리)
필기 JSON 삭제: deleteScoreAnnotations 호출
Promise.allSettled: 원본 파일과 필기 JSON 삭제를 병렬로 실행하고 두 작업이 모두 완료된 후 결과를 확인하여 하나라도 실패하면 오류를 던지도록 구현함

object-not-found 처리: deleteFileFromStorage에서 이미 처리하고 있으므로 변경 없이 성공으로 간주함
권한 오류 처리: Firebase Storage 삭제 실패 시 호출자에게 오류가 전파되고 Firestore 업데이트/삭제를 중단함
네트워크 오류 처리: 마찬가지로 삭제 실패 시 Firestore 업데이트/삭제를 중단하고 재시도 가능하게 함

악보 전체 삭제 순서:
1. 사용자 삭제 확인
2. 로그인 확인
3. pendingDeletes 등 개별 파일과 필기 JSON을 deleteScoreFileAssets로 삭제
4. 레거시 파일 삭제
5. 모든 삭제가 성공한 경우에만 Firestore 기록(deleteRecord) 삭제

개별 파일 삭제 순서:
1. 수정 사항 입력 완료
2. 로그인 확인
3. pendingDeletes 배열의 파일들을 deleteScoreFileAssets로 삭제
4. 삭제가 모두 성공한 경우에만 Firestore 배열 업데이트 (updateRecord)

Firestore 삭제 조건: 원본 파일과 필기 JSON 삭제 과정에서 오류(object-not-found 제외)가 전혀 발생하지 않았을 때만 삭제
Firestore files 배열 업데이트 조건: pendingDeletes에 있는 파일들의 원본과 필기 JSON 삭제가 모두 성공했을 때만 업데이트

삭제 실패 시 Firestore 기록: 삭제 실패 시 deleteRecord를 호출하지 않으므로 기존 기록 유지
삭제 실패 시 수정 모달: 수정 화면 닫힘 방지 및 pendingDeletes 상태 유지
삭제 실패 후 재시도: Firestore 기록과 files 배열이 유지되어 사용자가 다시 삭제 시도 가능

필기가 있는 PDF 전체 삭제: 원본 PDF, 필기 JSON, Firestore 기록 모두 삭제 성공
필기가 없는 PDF 전체 삭제: 원본 PDF 삭제 성공, 필기 JSON은 object-not-found로 성공 처리되어 최종 Firestore 기록 삭제
이미지 파일 삭제: 이미지 파일 삭제 성공, 필기 JSON은 object-not-found로 성공 처리되어 최종 Firestore 기록 삭제
개별 PDF 삭제: 선택한 PDF의 원본 및 필기 JSON만 삭제되고 Firestore files 배열 업데이트 성공
다른 파일 유지: 삭제되지 않은 다른 파일들의 Storage 및 Firestore 데이터는 정상 유지됨

원본 파일 경로: users/{uid}/repertoire/{repertoireId}/files/{fileName}
필기 JSON 경로: users/{uid}/repertoire/{repertoireId}/annotations/{fileId}/current.json (buildScoreAnnotationStoragePath 활용)

npm run build: 통과 (빌드 성공)
